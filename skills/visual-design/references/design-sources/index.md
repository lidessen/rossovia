# Design Source Index

Use this index only when host sources cannot resolve a material visual decision.
Select one to three complementary sources, then read only their detail files.
The index is fallback context, not a canon or style menu.

| Source | Select when the unresolved decision concerns | Detail |
|---|---|---|
| Apple HIG | adaptive content/control hierarchy, reading order, typography, semantic color across platform conditions | `apple.md` |
| Linear | product-wide balance, dense professional UI, redesign stress tests, generated themes | `linear.md` |
| IBM Carbon | content-led grids, spatial hierarchy, contextual layers and tokens | `carbon.md` |
| GitHub Primer | complex web-product focus, responsive layout, semantic token layers | `primer.md` |
| GOV.UK | public-service clarity, accessible reading form, small-screen content layout | `govuk.md` |

Useful combinations:

- Content-heavy public service: GOV.UK + Apple + one local cultural source.
- Dense professional product: Linear + Primer or Carbon.
- Color/token system: Primer + Carbon; add Apple for platform adaptation.
- Cross-surface redesign: Linear + one direct host analogue + one attractive
  counterexample.

For every selected source, record the observed effect, decision changed,
transfer boundary, and what must not be copied. Check the live official source
when freshness changes the decision.
