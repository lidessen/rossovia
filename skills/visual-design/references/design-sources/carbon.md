# IBM Carbon

## Use for

Content-led spatial structure, grids for different content archetypes,
hierarchy through scale/proximity/contrast, and contextual color layers.

## Decision lenses

- [2x Grid usage](https://carbondesignsystem.com/elements/2x-grid/usage/)
  begins from content, user goals, story, and desired action before selecting a
  layout model; it connects hierarchy, proximity, contrast, rhythm, and modules.
- [Color usage](https://carbondesignsystem.com/elements/color/usage/) shows
  contextual layer tokens and theme-aware component roles that express
  containment and elevation without component-owned raw values.

## Transfer boundary

Recover the movement from content type to spatial model and from semantic layer
to theme value. Do not assume a 16-column grid, 8-pixel unit, particular layer
count, IBM palette, or Carbon component treatment is intrinsically correct for
another project.
